## Commit structure
- Commit message must start with one of: `feat:`, `fix:`, `refactor:`
- First line must be a short summary under 50 characters
- After summary, add a newline followed by a detailed description

## Format rules
- Use markdown style
- Keep tone informal and concise
- Do not use phrases like “this commit” or “this change”
- Do not mention specific file names or code identifiers
- Each sentence in the description must be on a new line

## Content requirements
- List key changes clearly
- Highlight significant improvements or fixes
- Mention possible impact or usage implications
- Focus on what changed and why it matters, not implementation details
