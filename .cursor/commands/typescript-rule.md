- Prefer TypeScript, avoid any
- Use strong typing and proper interfaces
- Prefer type over interface when possible
- Use Zod for validation when needed

- Use functional components and hooks (React)
- Keep components small and reusable
- Use Redux Toolkit patterns correctly

- Use meaningful naming (camelCase, PascalCase)
- Use named exports over default exports
- Avoid hardcoding, use constants

- Prefer async/await over promises
- Handle errors properly

- Write clean, simple, maintainable code
- Follow SOLID principles
